# 2026-04-20=>【第三方仓管理】非AE平台直邮缺货订单库存释放逻辑优化

**一、需求背景（必填）**

**需求来源：**跨境业务运营中心 - AE 事业部

**适用部门：**跨境业务运营中心 - AE 事业部

**调研人群：**袁旭、黄明钦

**业务背景：**

当前多平台订单的库存释放逻辑存在差异，其中：

- AE 平台的 JIT /AE平台的直邮订单非 JIT 订单，已建立明确的库存优先级释放规则（如 AE平台JIT 订单优先用 AE 锁定库存、AE平台非 JIT 订单优先用 Temu 锁定库存），保障了 AE平台业务的库存资源利用率。
- 非 AE 平台直邮缺货订单当前仅支持直接触发**Temu 锁定库存**释放给直邮，缺乏库存不足时的没有兜底补充逻辑，影响订单处理效率。

**实现目标：**将非 AE 平台直邮订单的库存释放逻辑，调整为与 AE 平台非 JIT 订单的直邮订单一致的规则，具体目标如下：非AE平台的直邮缺货订单，**优先触发 Temu 锁定库存释放给直邮**；当 Temu 锁定库存数量不足、无法满足订单需求时，有条件的触发 AE 锁定接口释放直邮库存，补足缺口。

**二、详细描述（必填）**

**功能一：AE库存补货其他平台直邮缺货订单（非AE平台JIT订单）**

**所在路径：（选填）**

**实现步骤：**

1. AE库存释放给非AE平台直邮订单逻辑

非AE平台直邮缺货订单，优先触发temu锁定库存释放给直邮，若在temu锁定库存释放数量不够的情况下再触发AE锁定接口释放直邮库存。

- 规则：AE库存释放给直邮库存
- 触发节点：直邮仓出单后库存校验环节，若直邮仓在库库存<订单销量，则主动触发AE锁定库存释放流程
- **计算规则：仓发加权dms = dms7_cf\*0.4+dms14_cf\*0.4+dms30_cf\*0.2**
- 释放条件**：向下取整(AE库存/仓发加权dms) > max(交期,5)**，即 向下取整(AE库存支撑天数)>max(交期,5)
- 可释放库存数**：min（缺货库存数，向下取整(AE库存-仓发加权dms\* max(交期,5)) )**
- 仓发加权dms取值源

```
select product_id,dms_cf,update_time from (select *,
 
row_number() over(partition by product_id,left(update_time,10) order by update_time desc) rnk
 
from salepredict.ae_tuoguan_dms a
 
where left(update_time,10)=curdate()
 
) t where t.rnk = 1;
```
